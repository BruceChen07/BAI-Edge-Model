# 分阶段开发规划文档

## 1. 目标
基于 `solution design document.md`，将项目拆分为 4 个可独立推进且可归档的开发阶段，每阶段均输出代码、配置、测试结果、阶段日志和归档清单。

## 2. 阶段划分

### Phase 1：基础骨架与工程底座
- 核心功能模块：前后端工程初始化、基础目录结构、配置系统、数据库基础层、统一 API 响应、系统信息/设置接口、开发日志与归档机制
- 前置条件：设计文档冻结；Node/Python 可用；确定归档目录为 `deliverables`
- 预计交付周期：1-2 周
- 阶段输出：前后端可运行骨架、基础页面、基础 API、数据库初始化、阶段日志模板、归档脚本/归档清单

### Phase 2：知识库与文档入库链路
- 核心功能模块：知识库 CRUD、文件上传、任务中心、文档元数据存储、基础解析链路、文本切块、索引占位实现
- 前置条件：Phase 1 完成并具备可运行后端和数据库
- 预计交付周期：2-3 周
- 阶段输出：知识库管理、上传接口、任务状态管理、文档/分块表落地、基础入库流程

### Phase 3：RAG 对话、导出与双语界面
- 核心功能模块：会话管理、流式问答、检索编排、引用溯源、Markdown/Word/Excel 导出、前端中英文双语界面
- 前置条件：Phase 2 完成并具备可查询的文档分块数据
- 预计交付周期：2-3 周
- 阶段输出：聊天页面、SSE 对话接口、导出接口与文件生成、双语页面和基础预览

### Phase 4：记忆、Agent 与可追溯交付
- 核心功能模块：长期记忆 CRUD、Agent 工具注册与执行轨迹、开发日志累计、阶段归档自动化、最终交付说明
- 前置条件：Phase 3 完成并具备会话、检索和导出基础能力
- 预计交付周期：2-3 周
- 阶段输出：记忆页、Agent 页、工具调用链路、阶段归档产物、累计开发日志、最终交付清单

## 3. 过程管控要求
- 每个阶段完成后，将该阶段代码、配置、输出物同步归档到 `deliverables/phase-0X`
- 每个阶段必须更新 `deliverables/logs/development-log.md`
- 每个阶段必须补充测试清单与完成度量化结论
- 未完成项必须写明剩余工作量、阻塞原因和下一阶段承接关系

## 4. 开发日志标准模板
每个阶段日志必须包含：
1. 开发内容描述
2. 功能点覆盖测试用例
3. 开发完成度量化数据

## 5. 交付基线
- `solution design document.md`
- `development-plan.md`
- 前后端源码
- `deliverables/phase-01` ~ `deliverables/phase-04`
- `deliverables/logs/development-log.md`
